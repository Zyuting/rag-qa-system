# 系统总体架构 — PPT 要点

---

## 第1页：项目概况

**项目名称**：云南非物质文化遗产智能问答系统

**一句话描述**：基于 RAG（检索增强生成）技术的非遗知识库智能问答平台

**核心能力**：用户自然语言提问 → 系统从本地知识库检索相关文档 → 大模型生成回答

## 第3页：三层架构

### ① 数据层（知识库）
- 10个非遗项目的原始文档（TXT/PDF/DOCX等）
- BGE本地embedding模型
- FAISS向量索引文件
- 对话历史JSON文件

### ② 业务层（后端API）
- **RAG流水线**：文档加载 → 切分 → 编码 → 检索 → 生成
- **知识库管理**：创建、删除、上传文档、构建索引
- **对话管理**：多会话历史维护

### ③ 表现层（前端界面）
- 聊天对话界面、知识库切换、历史会话管理

## 第4页：一次问答的完整链路

```
用户提问 "东巴文化是什么？"
        │
        ▼
① API接收 → 验证知识库 + 索引是否存在
        │
        ▼
② 问题编码 → BGE模型转为768维向量
        │
        ▼
③ FAISS检索 → 向量索引中召回Top-20
        │
        ▼
④ MMR重排序 → 20个中精选Top-5（去重）
        │
        ▼
⑤ 组装Prompt → 历史对话 + 5篇参考 + 问题 + 要求
        │
        ▼
⑥ 调用千问LLM → 生成回答
        │
        ▼
⑦ 保存历史 → 存入JSON文件
        │
        ▼
⑧ 返回前端 → 回答文本 + 来源列表
```

## 第5页：两大核心流程

### 流程一：离线构建（一次性）
```
原始文档 → 加载 → 切分chunks → BGE编码 → FAISS索引
```
- 启动时运行 `build_kb.py`
- 10个知识库并行构建
- 产出索引文件供在线检索使用

### 流程二：在线问答（每次提问）
```
用户问题 → BGE编码 → FAISS检索 → MMR重排 → LLM生成 → 返回
```
- 实时处理用户提问
- 毫秒级检索，秒级回答
- 每个回答附带原文出处

## 第6页：技术栈

| 层次 | 技术 | 用途 |
|------|------|------|
| **后端框架** | FastAPI + Uvicorn | REST API服务 |
| **向量模型** | BGE-small-zh-v1.5 | 本地中文embedding（离线） |
| **向量数据库** | FAISS (IndexFlatIP) | 余弦相似度检索 |
| **大语言模型** | 通义千问 (DashScope) | 答案生成 |
| **文档解析** | PyPDF2 / python-docx | PDF/Word读取 |
| **数据格式** | NumPy / JSON | 向量运算 / 数据持久化 |
| **前端框架** | React + TypeScript + Vite | 用户界面 |
| **前端样式** | TailwindCSS + Framer Motion | 毛玻璃UI设计 |

## 第8页：系统亮点

| # | 亮点 | 说明 |
|---|------|------|
| 1 | **完整RAG流水线** | 7个模块端到端串联，从文档到答案全链路覆盖 |
| 2 | **本地embedding** | BGE模型本地运行，无需外部API，零成本 |
| 3 | **MMR去重机制** | 平衡相关性和多样性，避免返回重复内容 |
| 4 | **多知识库管理** | 支持10个独立非遗知识库，可动态增删文档 |
| 5 | **多会话支持** | 多条对话历史独立维护，可切换 |
| 6 | **来源可追溯** | 每个回答带原文文件名和相似度评分 |
| 7 | **一键启动** | start.bat全自动部署，开箱即用 |
