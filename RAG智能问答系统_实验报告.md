## 三、实验原理

### 3.1 RAG 技术概述

RAG（检索增强生成）是一种结合**信息检索**与**大语言模型生成**的技术范式。其核心思想是：在 LLM 回答用户问题之前，先从知识库中检索与问题相关的文档片段，将这些片段作为上下文提供给 LLM，使 LLM 能够基于真实、最新的知识生成回答，有效缓解大大语言模型生成**的技术范式。其核心思想是：在 LLM 回答用户问题之前，先从知识库中检索与问题相关的文档片段，将这些片段作为上下文提供给 LLM，使 LLM 能够基于真实、最新的知识生成回答，有效缓解大模型的"幻觉"问题。\n\n### 3.2 系统架构\n\n本系统采用前后端分离架构，整体流程分为**离线构建**和**在线推理**两个阶段：\n\n```\n┌─────────────────────────────────────────────────────────────┐\n│                    离线构建阶段（build_kb.py）                │\n│                                                             │\n│  原始文档 → 切分(500字符/chunk) → bge-small-zh 编码(384维) → FAISS 索引 │\n└─────────────────────────────────────────────────────────────┘\n                              ↓\n┌─────────────────────────────────────────────────────────────┐\n│                    在线推理阶段（rag_01_app.py）                │\n│                                                             │\n│  用户问题 → Embedding 编码 → FAISS 粗排 Top-20 → MMR 重排序 Top-5 │\n│      → 构建 Prompt(参考+历史+规则) → 千问 API 生成 → 返回答案+来源  │\n└─────────────────────────────────────────────────────────────┘\n```\n\n### 3.3 向量检索原理\n\n本系统使用 FAISS 的 `IndexFlatIP`（内积索引）。向量经过 L2 归一化后，内积等价于余弦相似度，值域为 [-1, 1]，越接近 1 表示语义越相似。\n\n### 3.4 MMR 重排序原理\n\nMMR（Maximal Marginal Relevance）在相关性和多样性之间取得平衡，公式如下：\n\n```\nMMR = λ × Sim(query, doc) - (1-λ) × max(Sim(doc, already_selected))\n```\n\n- **λ = 0.7**：偏向相关性\n- **多样性惩罚**：候选文档与已选中文档的最大相似度\n- 效果：结果既相关又覆盖不同方面，避免多条相似内容\n\n### 3.5 Embedding 模型\n\n使用 **BAAI/bge-small-zh-v1.5** 作为文本向量编码模型：\n- 输出 384 维向量\n- 专为中文语义匹配优化\n- 模型体积小（约 33MB），适合本地 CPU 部署\n\n---\n\n## **多知识库管理**：支持 11 个非遗主题知识库的独立存储、构建与检索\n2. **高效的向量检索**：FAISS 384 维向量索引，毫秒级检索响应\n3. **MMR 重排序**：平衡结果的相关性与多样性，避免内容重复\n4. **智能拒答机制**：基于 0.40 相似度阈值，有效防止跨库噪声回答\n5. **多轮对话管理**：支持会话历史持久化，维持对话连贯性\n6. **来源追溯**：每个回答可追溯到原始文档片段，具备可验证性\n7. **前后端分离架构**：FastAPI 后端 + React 前端，接口清晰、可扩展\n\n### 7.2 存在的不足\n\n1. **拒绝阈值硬编码**：0.40 的拒答阈值目前在代码中硬编码，应抽离为可配置参数\n2. **缺乏反馈闭环**：用户点赞/点踩数据仅记录到日志，未用于优化检索结果\n3. **LLM 补充知识无法精确控制**：Prompt 中限制补充不超过 30%，但 LLM 不保证严格遵循\n4. **仅支持 CPU 检索**：FAISS 使用 CPU 版本，大规模场景下可启用 GPU 加速\n\n### 7.3 改进方向\n\n1. **动态阈值**：根据每个知识库的历史检索分数分布自动计算拒答阈值\n2. **反馈驱动优化**：利用用户反馈数据微调 Embedding 模型或调整检索权重\n3. **混合检索**：结合关键词检索（BM25）与向量检索，提升召回率\n4. **流式输出**：支持 SSE 流式返回 LLM 生成结果，改善用户体验\n\n---\n\n## 八、参考文献\n\n1.
