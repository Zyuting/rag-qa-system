# 数据处理与知识库构建 — PPT 要点

---

## 第1页：数据处理与知识库构建流程

**一句话概括**：原始文档 → 切碎成块 → 转成向量 → 建索引

**四大步骤**：

| 步骤 | 做什么 | 输入 → 输出 |
|------|--------|------------|
| ① 文档加载 | 读取目录下所有文件 | TXT/PDF/DOCX/MD/HTML → 纯文本列表 |
| ② 文档切分 | 按段落拆成小块 | 长文本 → 500字/块的 chunks |
| ③ 向量编码 | BGE模型转成向量 | chunks → 768维向量 |
| ④ 构建索引 | FAISS建索引库 | 向量 → 可检索的索引文件 |

---

## 第2页：关键参数（记住这4个数）

| 参数 | 值 | 说明 |
|------|-----|------|
| chunk_size | **500字符** | 每个文档块大小 |
| chunk_overlap | **100字符** | 相邻块重叠，防语义断层 |
| 向量维度 | **768维** | BGE-small模型输出 |
| batch_size | **32条** | 编码时每批处理数量 |

---

## 第3页：文档切分细节

- **段落级切分**：按换行符拆段落，以段落为单位拼接
- **不超过500字**：段落累加到500字时截断
- **重叠100字**：新chunk带上上一段末尾100字
- **超长段落**：超过500字的段落，滑动窗口切分

---

## 第4页：向量编码与索引构建

**向量编码**：
- 使用 **BGE-small-zh-v1.5** 本地模型（完全离线，无需API）
- 全局单例懒加载，首次用才加载
- 批量编码，32条一批

**索引构建**：
- 向量做 **L2归一化**（内积 = 余弦相似度）
- 使用 **FAISS IndexFlatIP**（内积索引）
- 最终产出2个文件：
  - `faiss_index.index` — 二进制索引
  - `faiss_metadata.json` — 原文信息

---

## 第5页：知识库数据量

| 项目 | 数量 |
|------|------|
| 知识库总数 | **10个**非遗项目 |
| 每个库原始文档 | **5篇** |
| 文档总数 | **50篇** |
| 切分后总chunks | **约500~700块** |
| 向量维度 | **768维** |

---

## 第6页：三种构建方式

| 方式 | 命令/接口 | 适用场景 |
|------|-----------|---------|
| 一键全量构建 | `python build_kb.py` | 首次部署 |
| 单库构建 | `POST /api/kb/build` | 更新某个知识库 |
| 上传后自动重建 | 上传时 `auto_rebuild=true` | 新增文档后自动更新 |
