### 3.3 FAISS 向量检索原理

#### 1）向量检索的基本思想

文本经过 Embedding 模型编码后，被映射到高维语义空间中的一点。语义相近的文本在该空间中距离也相近。因此，检索问题转化为：**给定一个查询向量，在向量库中找到与其最相似的 k 个向量**。

#### 2）余弦相似度

本系统使用**余弦相似度**衡量文本之间的语义相似度，取值范围 [-1, 1]，值越大表示语义越相似：

$$sim(A, B) = \cos(\theta) = \frac{A \cdot B}{|A| \times |B|}$$

为提高计算效率，系统在构建索引时对所有向量做 **L2 归一化**，使向量长度变为 1。归一化后，余弦相似度等价于 **内积（Inner Product）**，无需再计算分母，大幅简化运算（对应代码 `rag_05_faiss_goujian.py` 第 52-58 行）：

```python
# L2 归一化 → 内积等价于余弦相似度
faiss.normalize_L2(vectors)

dim = vectors.shape[1]  # 512 维
index = faiss.IndexFlatIP(dim)  # Inner Product（余弦相似度）
index.add(vectors)
```

#### 3）搜索过程

查询时，将用户问题编码为同维度向量并做 L2 归一化，然后调用 FAISS 的 `search` 方法完成检索（对应代码 `rag_06_jiansuo_mmr.py` 第 98-105 行）：

```python
# 编码查询向量
query_vec = encoder.encode([query], convert_to_numpy=True).astype(np.float32)
faiss.normalize_L2(query_vec)

# FAISS 搜索：返回 Top-fetch_k 个结果
actual_fetch_k = min(fetch_k, index.ntotal)
scores, indices = index.search(query_vec, actual_fetch_k)
```

`index.search` 底层执行过程如下：

1. 计算查询向量与库中每个向量的内积（即余弦相似度）
2. 将得分从高到低排序
3. 返回得分最高的前 `fetch_k` 个结果及其对应的索引位置

#### 4）IndexFlatIP —— 精确搜索

系统使用 `IndexFlatIP` 索引类型，其检索方式为**暴力全量搜索**（Brute Force）。当查询到来时，逐一计算查询向量与库中**所有**向量的内积，找出得分最高的前 k 个结果。

| 特性 | 说明 |
|------|------|
| **索引类型** | IndexFlatIP（内积索引） |
| **搜索方式** | 精确搜索（暴力搜索） |
| **优点** | 结果精确，无信息损失 |
| **缺点** | 数据量大时耗时线性增长 |
| **适用场景** | 中小规模数据集（万级以下） |

本系统单个知识库的向量规模在几百到几千条量级，使用精确搜索即可在毫秒级完成，因此无需引入 IVF、HNSW 等近似索引。

#### 6）阈值过滤

系统设计了双层的分数控制机制：

- **最低阈值**（`MMR_SCORE_THRESHOLD = 0.01`）：在 `mmr_search` 内部过滤掉完全无关的结果（`rag_06_jiansuo_mmr.py` 第 128-136 行）
- **拒答阈值**（`score < 0.40`）：在 `rag_01_app.py` 的 `/api/chat` 接口中判断，低于 0.40 视为不相关，触发拒答

通过这一机制，有效防止了跨知识库的噪声响应，是系统可靠性的关键设计。
